## 既存アプリへの不満が爆発した

筋トレが趣味で、今までいくつかのトレーニング記録アプリを使ってきた。
でも、どれもこれも微妙にストレスが溜まる。

**具体的に何が不満だったか：**

- **3ヶ月前の履歴を見ようとすると課金を求められる** — 自分のデータなのに？
- **広告がうざい** — セット間の休憩中に全画面広告が出てくる
- **データのエクスポートに制限がかかる** — 自分の記録を自由に取り出せない
- **スロートレーニングしたいのに、テンポやBPMを数える機能がない** — メトロノーム欲しい
- **休憩時間・筋トレ時間を記録できない** — 振り返りたいのに
- **「腕立て伏せ」で一括り** — ナロー、ワイド、ダイヤモンド...全部違うんだが？
- **有酸素・スタジオレッスンのトータル時間を把握できない** — BODYCOMBATやBODYPUMPを週に何回、合計何時間やったか知りたいのに

ないなら作るか、ということで作った。

## 作ったもの

### 主な機能

| 機能 | 説明 |
|------|------|
| **バリエーション記録** | 腕立て伏せでも「ナロー」「ワイド」「ブルガリアン右」など細かく記録可能 |
| **テンポ記録** | 「2-1-2」のようなテンポ表記でスロトレの速度を管理 |
| **メトロノーム** | BPM設定でスロートレーニングのリズムをキープ |
| **インターバルタイマー** | セット間の休憩時間を記録 |
| **ワークアウトテンプレート** | よく使うメニューをテンプレ化してワンタップで呼び出し |
| **カレンダー表示** | 日別のトレーニング履歴をカレンダーで一覧 |
| **完全ローカル保存** | アカウント不要。データは端末内に保存。課金も広告もなし |
| **カスタム種目** | 自分だけの種目をアイコン・カラー付きで自由に追加 |
| **JSONエクスポート** | AI分析に最適化された形式でデータをエクスポート |

### 既存アプリとの違い

```
既存アプリ：腕立て伏せ → 10回 × 3セット（以上）
SlowRep  ：腕立て伏せ → ナロー / テンポ2-1-2 / 10回 × 3セット / BPM60
```

この差が、スロートレーニングをやっている人には大きい。

## Claude Code で1週間で作った

今回の開発は **Claude Code の練習**も兼ねていた。

実際にやってみた感想：

- **UIの実装が爆速** — 「こういう画面が欲しい」と伝えるだけで、React Native + Paper のコンポーネントを組み立ててくれる
- **状態管理の設計** — Zustand のストア設計もほぼ Claude Code に任せた。persist ミドルウェアとの連携もスムーズ
- **型定義** — TypeScript の型定義を先に作ってもらうと、その後の実装がブレない
- **デバッグ** — エラーメッセージを貼るだけで原因特定→修正まで一気にやってくれる

### 開発の進め方

1. まず型定義（`types/workout.ts`）を固める
2. ストア（Zustand）を作る
3. 画面を1つずつ実装
4. 動作確認 → フィードバック → 修正のループ

Claude Code との開発で意識したのは、**最初に型をしっかり定義すること**。型があると Claude Code の出力精度がかなり上がる。

### 開発期間の内訳（ざっくり）

- 1-2日目：型定義 + ストア設計 + 基本画面
- 3-4日目：テンポ記録・メトロノーム・タイマー実装
- 5-6日目：テンプレート機能・カスタム種目・UI調整
- 7日目：EAS Build → TestFlight 提出

## データ構造

参考までに、コアとなるデータ構造：

```typescript
interface SetEntry {
  reps: number;
  weight?: number;       // 重量（kg）
  variation?: string;    // バリエーション（ナロー, ワイドなど）
  tempo?: string;        // テンポ表記（"2-1-2"）
  assistance?: boolean;  // 補助あり/なし
}

interface WorkoutSet {
  entries: SetEntry[];
  completed: boolean;
  metronome?: boolean | MetronomeSettings;
}

interface Exercise {
  id: string;
  type: ExerciseType;
  name: string;
  sets: WorkoutSet[];
}

interface DailyWorkout {
  date: string; // YYYY-MM-DD
  exercises: Exercise[];
  durationSeconds?: number;
  timerRecords?: TimerRecord[];
}
```

`SetEntry` に `variation` と `tempo` を持たせているのがポイント。
同じ「腕立て伏せ」でも、セットごとにバリエーションやテンポを変えられる。

## JSONエクスポート — AIに自分のトレーニングを分析させる

個人的に一番こだわった機能がこれ。

トレーニングデータをJSON形式でエクスポートできるようにした。なぜJSONかというと、**ChatGPTやClaudeにそのまま食わせて分析できる**から。

例えばエクスポートしたJSONをAIに渡して：

- 「先月と今月でボリュームはどう変化した？」
- 「オーバートレーニングの兆候はある？」
- 「テンポ4-2-2と2-1-2、どっちがレップ数伸びてる？」

みたいな質問ができる。

既存アプリだとCSVエクスポートすら課金が必要だったり、エクスポート自体に制限がかかっていたりする。**自分のデータなのに自由に使えない**のは本当にストレスだった。

SlowRepは全データがローカルに保存されていて、JSONエクスポートも無制限。自分のデータは自分のもの。
