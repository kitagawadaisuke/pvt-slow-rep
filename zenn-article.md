## 既存アプリへの不満が爆発した

筋トレが趣味で、今までいくつかのトレーニング記録アプリを使ってきた。
でも、どれもこれも微妙にストレスが溜まる。

**具体的に何が不満だったか：**

- **3ヶ月前の履歴を見ようとすると課金を求められる** — 自分のデータなのに？
- **広告がうざい** — セット間の休憩中に全画面広告が出てくる
- **データのエクスポートに制限がかかる** — 自分の記録を自由に取り出せない
- **スロートレーニングしたいのに、テンポやBPMを数える機能がない** — メトロノーム欲しい
- **休憩時間・筋トレ時間を記録できない** — 振り返りたいのに
- **「腕立て伏せ」で一括り** — ナロー、ワイド、ダイヤモンド...全部違うんだが？
- **有酸素・スタジオレッスンのトータル時間を把握できない** — BODYCOMBATやBODYPUMPを週に何回、合計何時間やったか知りたいのに

ないなら作るか、ということで作った。

## 作ったもの

**SlowRep** — スロートレーニングに特化したワークアウト記録アプリ。

### 主な機能

| 機能 | 説明 |
|------|------|
| **バリエーション記録** | 腕立て伏せでも「ナロー」「ワイド」「ブルガリアン右」など細かく記録可能 |
| **テンポ記録** | 「2-1-2」「4-2-2」のようなテンポ表記でスロトレの速度を管理 |
| **メトロノーム** | BPM設定でスロートレーニングのリズムをキープ。拍子切替・セットカウント対応 |
| **インターバルタイマー** | セット間の休憩時間を管理。音声カウントダウン付き |
| **メニュー管理** | よく使うメニューを保存してワンタップで呼び出し。種目・回数・重量・テンポの編集も可能 |
| **カレンダー表示** | 日別のトレーニング履歴をカレンダーで一覧 |
| **完全ローカル保存** | アカウント不要。データは端末内に保存。課金も広告もなし |
| **カスタム種目** | 自分だけの種目をアイコン・カラー付きで自由に追加 |
| **JSONエクスポート** | AI分析に最適化された形式でデータをエクスポート |
| **統計ダッシュボード** | 月間・年間のトレーニング日数、総セット数、筋トレ時間を一目で確認 |

### 既存アプリとの違い

```
既存アプリ：腕立て伏せ → 10回 × 3セット（以上）
SlowRep  ：腕立て伏せ → ナロー / テンポ4-2-2 / 10回 × 3セット / BPM60
```

この差が、スロートレーニングをやっている人には大きい。

### 音声合成（自前実装）

メトロノームのクリック音とインターバルタイマーの終了音は、外部の音声ファイルを使わず **WAVデータをJavaScriptで動的に生成**している。

- 通常ビープ: 620Hz / 35ms / 指数減衰エンベロープ
- アクセントビープ: 900Hz / 45ms
- タイマー終了音: C5→G5の2音ベル（柔らかいアタック + 自然減衰 + 倍音）

8ビットPCM WAVのヘッダーからサンプルデータまで全てコードで生成し、Base64エンコードして `expo-av` で再生している。

## Claude Code で1週間で作った

今回の開発は **Claude Code の練習**も兼ねていた。

実際にやってみた感想：

- **UIの実装が爆速** — 「こういう画面が欲しい」と伝えるだけで、React Native + Paper のコンポーネントを組み立ててくれる
- **状態管理の設計** — Zustand のストア設計もほぼ Claude Code に任せた。persist ミドルウェアとの連携もスムーズ
- **型定義** — TypeScript の型定義を先に作ってもらうと、その後の実装がブレない
- **デバッグ** — エラーメッセージを貼るだけで原因特定→修正まで一気にやってくれる
- **UIリデザイン** — 「App Storeにありそうな高品質なUI」とプロンプトを投げたら、デザインシステムの提案からスタイル実装まで一括でやってくれた
- **パフォーマンス最適化** — FlashList導入、コンポーネント分割、Zustand selector最適化まで自動で提案・実装

### 開発の進め方

1. まず型定義（`types/workout.ts`）を固める
2. ストア（Zustand）を作る
3. 画面を1つずつ実装
4. 動作確認 → フィードバック → 修正のループ
5. UIリデザイン → パフォーマンス最適化
6. EAS Build → TestFlight → App Store 審査提出

Claude Code との開発で意識したのは、**最初に型をしっかり定義すること**。型があると Claude Code の出力精度がかなり上がる。

### App Store 審査のポイント

初回審査では **Guideline 2.1 - Information Needed** で追加情報を求められた。対応として：

- 実機での画面収録動画を添付
- アプリの目的、アクセス方法、外部サービスの有無を英語で説明
- 地域差なし、規制産業でないことを明記

これらをApp Store Connectのメモ欄に記載して再提出したら通った。初めてのApp Store公開の人は参考にしてほしい。

## データ構造

参考までに、コアとなるデータ構造：

```typescript
interface SetEntry {
  reps: number;
  weight?: number;       // 重量（kg）
  variation?: string;    // バリエーション（ナロー, ワイドなど）
  tempo?: string;        // テンポ表記（"2-1-2"）
  assistance?: boolean;  // 補助あり/なし
}

interface WorkoutSet {
  entries: SetEntry[];
  completed: boolean;
  metronome?: boolean | MetronomeSettings;
}

interface Exercise {
  id: string;
  type: ExerciseType;
  name: string;
  sets: WorkoutSet[];
}

interface DailyWorkout {
  date: string; // YYYY-MM-DD
  exercises: Exercise[];
  durationSeconds?: number;
  timerRecords?: TimerRecord[];
}
```

`SetEntry` に `variation` と `tempo` を持たせているのがポイント。
同じ「腕立て伏せ」でも、セットごとにバリエーションやテンポを変えられる。

## JSONエクスポート — AIに自分のトレーニングを分析させる

個人的に一番こだわった機能がこれ。

トレーニングデータをJSON形式でエクスポートできるようにした。なぜJSONかというと、**ChatGPTやClaudeにそのまま食わせて分析できる**から。

例えばエクスポートしたJSONをAIに渡して：

- 「先月と今月でボリュームはどう変化した？」
- 「オーバートレーニングの兆候はある？」
- 「テンポ4-2-2と2-1-2、どっちがレップ数伸びてる？」

みたいな質問ができる。

既存アプリだとCSVエクスポートすら課金が必要だったり、エクスポート自体に制限がかかっていたりする。**自分のデータなのに自由に使えない**のは本当にストレスだった。

SlowRepは全データがローカルに保存されていて、JSONエクスポートも無制限。自分のデータは自分のもの。

## 実際にかかった期間

正直に書くと、**大枠は1週間で作れた。でもそこからが長かった。**

Claude Codeを使えばUIの実装やロジックの組み立ては爆速で進む。最初のプロトタイプは本当に数日でできた。

でも実際に毎日使ってみると、「ここのボタン押しにくいな」「メトロノームのカウントタイミング違うな」「テンプレートの編集が今日の記録に影響するのはダメだろ」みたいな細かい不満が次々と出てくる。

結局、**細かいアップデートを何十回も繰り返して、App Storeに公開するまでに約1ヶ月かかった。**

まあ、そりゃそうだよなという話。アプリ開発の8割は「最後の2割の磨き込み」に使われる。Claude Codeはその磨き込みのスピードも上げてくれたけど、やること自体は減らない。

それでも、1ヶ月で個人開発アプリをApp Storeに公開できたのは、Claude Codeのおかげだと思う。

## まとめ

- 既存の筋トレアプリ、課金・広告・機能制限が多すぎる
- スロートレーニングに必要な「テンポ」「メトロノーム」「バリエーション」を記録できるアプリがなかった
- Claude Code を使えば大枠は1週間で作れる。でも磨き込みに1ヶ月かかった
- データは完全ローカル。課金なし。広告なし。自分のデータは自分のもの

**「ないなら作る」の精神で、満足いくものができた。**
